simulator

simulator.py - produce communcication traces for machine learning applications.

Use this tool to:

  • Generate PCAP files based on transmitter activity

comms_ml.simulator.collect_features(pcap_dir, sampling_dict, progress_max_time=None)

Returns the features collected from the PCAPs in pcap_dir, using the given sampling configuration

comms_ml.simulator.create_nodes(node_descs, sim_time_s=None, message_cache=None)

Returns a list of transmitters based on the given configs

comms_ml.simulator.get_features(status_changes, n_tx, timescale_s=0.1, n_channels=None, tx_map=None)

Generates feature vectors from activity trace. Initially made to collect average IAT for each channel. tx_map (optional) is {tx_index : channel_index}

comms_ml.simulator.packets_from_dir(pcap_dir)

Iterates over every packet in every (sorted) PCAP in the given directory

comms_ml.simulator.play_status_changes(status_changes, tx_list)

An animated visual representation

comms_ml.simulator.plot_status_timeline(status_changes, n_tx)

A static visual representation

comms_ml.simulator.run_sim(sim_dict, init_only=False)

Executes the simulator’s finite events as defined in the given config

If init_only is set, the simulator performs a “dry run” leaving the necessary init for sampling.
